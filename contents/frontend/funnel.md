## 2. 퍼널을 선택한 이유

어떤 문제를 해결하기 위해 퍼널을 도입했는지?

- 선형적인 단계가 있는 프로세스
- 이전 단계로의 되돌아가기가 필요한 경우
- 전체 프로세스의 컨텍스트 관리가 중요한 경우

1차, 2차 3차 배포에 따라 순차적인 퍼널 단계 추가

사용자 경험 측면의 장점

- 복잡한 프로세스를 작은 단계들로 나누어 사용자가 쉽게 이해하고 완료할 수 있게 합니다
- 한 번에 너무 많은 정보를 요구하지 않아 사용자의 인지 부하를 줄일 수 있습니다

유지보수

- 각 단계별로 독립적인 컴포넌트를 만들 수 있어 코드 관리가 용이합니다
- 새로운 단계를 추가하거나 제거하기 쉽습니다

내가 겪은 문제점을 좀 더 자세하게 적기

- 모바일을 고려했을 때 너무나도 긴 스크롤

## 3. 구현 코드

어떤 것을 신경썼는지?

- 이전/다음 단계 데이터 유지
- Context 사용 이유?
- 실제로 적용 과정 중 이슈는?

구현 과정 중 어려움을 자세하게 적기

- 이전/다음 상태 값 보존 방법
- 적절한 메서드 제공

지향

- 코드를 한 곳에서 한눈에 파악할 수 있으면 좋겠다.
- 각 스텝 별로 상태를 정의해서 사용할 수 있고 히스토리별로 해당 스텝의 상태를 보장할 수 있다.
